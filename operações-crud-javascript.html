Tutorial: Sistema de Gestão de Biblioteca com Node.js e MySQL

** ATENÇÃO **: para essa prática, será necessário executar o script de banco de dados no arquivo biblioteca.sql para criar as tabelas e inserir os dados iniciais.

1. Operações CRUD

CRUD é um acrônimo que representa as quatro operações básicas de manipulação de dados:

  • Create (Criar): Inserir novos registros no banco de dados
  • Read (Ler): Consultar e exibir dados existentes
  • Update (Atualizar): Modificar registros existentes
  • Delete (Deletar): Remover registros do banco de dados

Exemplo no Sistema de Biblioteca:

  • Create: Adicionar um novo livro
  • Read: Listar todos os livros ou visualizar detalhes de um livro
  • Update: Editar informações de um livro existente
  • Delete: Remover um livro do sistema

2. Protocolo HTTP e Verbos HTTP

O HTTP (HyperText Transfer Protocol) é o protocolo de comunicação usado na web. Os verbos HTTP definem o tipo de ação que queremos realizar:

Principais Verbos HTTP:

GET

  • Propósito: Recuperar dados do servidor
  • Características: Seguro, idempotente, não modifica dados
  • Exemplo: GET /livros - busca todos os livros

POST

  • Propósito: Enviar dados para criar novos recursos
  • Características: Não é idempotente, modifica o estado do servidor
  • Exemplo: POST /livros - cria um novo livro

PUT

  • Propósito: Atualizar um recurso existente completamente
  • Características: Idempotente, substitui o recurso inteiro
  • Exemplo: PUT /livros/1 - atualiza o livro com ID 1

DELETE

  • Propósito: Remover um recurso do servidor
  • Características: Idempotente, remove o recurso
  • Exemplo: DELETE /livros/1 - remove o livro com ID 1

Mapeamento CRUD → HTTP:

Operação CRUD Verbo HTTP Exemplo de Rota
Create POST POST /livros
Read GET GET /livros
Update PUT PUT /livros/1
Delete DELETE DELETE /livros/1

3. Conexão com Banco de Dados MySQL

Para conectar uma aplicação Node.js ao MySQL, você precisa:

Requisitos:

  1. Servidor MySQL instalado e rodando
  2. Banco de dados criado
  3. Credenciais de acesso (usuário e senha)
  4. Driver MySQL para Node.js (mysql2)

Configuração da Conexão: 

const mysql = require('mysql2');

const connection = mysql.createConnection({
    host: 'localhost',        // Endereço do servidor MySQL
    user: 'root',            // Usuário do banco
    password: 'sua_senha',   // Senha do usuário
    database: 'biblioteca'   // Nome do banco de dados
});

// Testar conexão
connection.connect((err) => {
    if (err) {
        console.error('Erro ao conectar:', err);
        return;
    }
    console.log('Conectado ao MySQL!');
});

Executando Consultas: 

// SELECT
connection.query('SELECT * FROM livro', (err, results) => {
    if (err) throw err;
    console.log(results);
});

// INSERT
connection.query(
    'INSERT INTO livro (titulo, ano_publicacao) VALUES (?, ?)',
    ['Meu Livro', 2023],
    (err, result) => {
        if (err) throw err;
        console.log('Livro inserido com ID:', result.insertId);
    }
);

5. Explicação dos Pacotes

5.1 Express 

const express = require('express');
const app = express();
  • Função: Framework web para Node.js
  • Uso: Criar servidor HTTP, definir rotas, gerenciar middlewares
  • Benefícios: Simplifica criação de APIs e aplicações web

5.2 Method-Override 

const methodOverride = require('method-override');
app.use(methodOverride('_method'));
  • Função: Permite usar verbos HTTP PUT e DELETE em formulários HTML
  • Uso: HTML só suporta GET e POST nativamente
  • Implementação: Adiciona ?_method=PUT ou ?_method=DELETE na URL

5.3 EJS (Embedded JavaScript) 

app.set('view engine', 'ejs');
  • Função: Engine de template para gerar HTML dinâmico
  • Uso: Inserir dados JavaScript em páginas HTML
  • Sintaxe: <%= variavel %> para exibir dados, <% codigo %> para lógica

5.4 Body-Parser 

const bodyParser = require('body-parser');
app.use(bodyParser.urlencoded({ extended: true }));
  • Função: Middleware para processar dados de formulários
  • Uso: Converte dados POST em objeto JavaScript acessível via req.body
  • Tipos: Suporta URL-encoded e JSON

5.5 MySQL2 

const mysql = require('mysql2');
  • Função: Driver para conectar Node.js ao MySQL
  • Uso: Executar consultas SQL, gerenciar conexões
  • Vantagens: Suporte a Promises, melhor performance que mysql

7. Fluxo de Funcionamento do Sistema

7.1 Inicialização:

  1. Node.js carrega o app.js
  2. Express cria o servidor HTTP
  3. Conexão com MySQL é estabelecida
  4. Middlewares são configurados
  5. Rotas são definidas
  6. Servidor fica escutando na porta 3000

7.2 Processamento de Requisições:

  1. Cliente faz requisição HTTP
  2. Express identifica a rota correspondente
  3. Middleware processa dados (body-parser, method-override)
  4. Controlador executa a lógica de negócio
  5. MySQL executa consultas no banco
  6. EJS renderiza a página com os dados
  7. Resposta é enviada ao cliente

8. Exemplo Prático: Criar um Livro

8.1 Formulário HTML (criar.ejs): 

<form action="/livros" method="POST">
    <input type="text" name="titulo" required>
    <input type="number" name="ano_publicacao">
    <button type="submit">Criar Livro</button>
</form>

8.2 Rota no Express (app.js): 

app.post('/livros', (req, res) => {
    const { titulo, ano_publicacao } = req.body;
    
    const query = 'INSERT INTO livro (titulo, ano_publicacao) VALUES (?, ?)';
    
    connection.query(query, [titulo, ano_publicacao], (err, result) => {
        if (err) {
            console.error('Erro:', err);
            return res.status(500).send('Erro interno');
        }
        res.redirect('/livros');
    });
});

8.3 Fluxo Completo:

  1. Usuário preenche formulário e clica "Criar Livro"
  2. Browser envia POST para /livros
  3. body-parser extrai dados do formulário
  4. Rota POST executa query INSERT no MySQL
  5. Após sucesso, redireciona para lista de livros
  6. Lista atualizada é exibida ao usuário
